# Em+ — Calm Care UI Direction

> This document translates the "calm, minimal, conversion-focused" qualities observed in LookAway into Em+'s product and brand context.
>
> It does **not** replace [`VISUAL_DESIGN_GUIDE.md`](VISUAL_DESIGN_GUIDE.md). Instead, it narrows that system into a more usable direction for the first user-facing surfaces: marketing landing, onboarding, and home.

## 0. Method & Inputs

This direction came from combining two inputs:

1. **YC-style startup idea validation**
2. **LookAway landing/style analysis**

The YC prompt was used to pressure-test Em+'s product story before touching visuals.

Questions that mattered most:

- is the problem real and recognized by users?
- what do users do today instead?
- what is the first believable wedge?
- what can the user do right now?

That pushed Em+ away from a broad "relationship super app" presentation and toward a clearer first promise:

> help people remember the right things and care at the right moment

LookAway then informed **how that promise should be presented**:

- calm first impression
- clear headline hierarchy
- strong spacing and quiet sections
- believable feature explanation
- trust through clarity, not spectacle

This is therefore **not** just a visual refresh.

It is a product-positioning translation:

- **YC** defines what needs to be clear
- **LookAway** informs how that clarity should feel on-screen

### Source notes

- LookAway reference: [lookaway.com](https://lookaway.com/)
- YC messaging reference: [Practical Design: Messaging](https://www.ycombinator.com/blog/practical-design-messaging)

## 1. Why This Direction Exists

Em+ currently communicates as:

- warm
- emotional
- premium
- visually rich

That is not wrong. The problem is that the product can become:

- too decorative before it becomes clear
- too feature-heavy before it feels useful
- too "romantic app" before it feels trustworthy

The goal of this direction is to shift Em+ toward:

**calm romantic utility**

Meaning:

- still warm
- still soft
- still emotionally aware
- but much clearer, calmer, and more action-oriented

## 2. North Star

Em+ should feel like:

> a gentle assistant that helps you remember the right things and care at the right moment

It should **not** feel like:

- a dreamy relationship dashboard
- a feature showroom
- a social app for couples
- a luxury glassmorphism demo

## 3. Core Design Principles

### 3.1 Problem-first, not feature-first

Every key surface should explain the user's pain before showing product breadth.

Use:

- "Hay quên những điều quan trọng?"
- "Không biết nên nhắn gì đúng lúc?"
- "Sắp đến dịp quan trọng nhưng chưa chuẩn bị?"

Avoid:

- opening with architecture
- opening with all modules
- opening with "timeline + budget + care + live sync"

### 3.2 Calm before romance

Visual calm is now the default posture.

That means:

- brighter light surfaces
- less blur
- less glow
- fewer stacked gradients
- fewer floating decorative elements

Romance still exists, but as tone and warmth, not as visual noise.

### 3.3 One primary action per screen

Each screen should answer:

- what matters most here?
- what should the user do next?

If a screen contains multiple strong actions, hierarchy is broken.

### 3.4 Trust over spectacle

Em+ deals with intimate data. Users should feel:

- safe
- guided
- not manipulated

So:

- surfaces should be steady and readable
- copy should be human, not hype-heavy
- empty states should be useful, not ornamental

### 3.5 Emotional restraint

Em+ should feel caring, not sugary.

Prefer:

- "Một điều nhỏ cho hôm nay"
- "Điều quan trọng sắp tới"
- "Những gì nên nhớ về người ấy"

Avoid:

- overly dramatic love language
- heavy hearts / sparkles everywhere
- fantasy-like framing on utilitarian screens

## 4. Brand Translation

We are **not** adopting LookAway's exact palette or brand neutrality.

We **are** borrowing its discipline.

### Keep from current Em+

- Be Vietnam Pro typography
- warm taupe neutrals
- coral as emotional brand signal
- large radii
- gentle language

### Reduce from current Em+

- always-on dark-first presentation
- high blur glass on most panels
- overuse of coral + indigo + teal together
- decorative motion on conversion surfaces
- image-heavy feature storytelling

### Borrow from LookAway

- light-first web presentation
- large, clear value proposition
- quiet sections with strong spacing
- visual explanation over dense copy
- fewer features, explained better

## 5. Color & Surface Rules

### 5.1 Mode strategy

- **Web marketing:** light-first
- **Mobile utility screens:** both modes supported, but light-mode composition becomes the reference
- **Dark mode:** supported, but should still feel calm and readable rather than glossy

### 5.2 Palette role

- **Coral:** primary brand and CTA
- **Taupe / warm off-white:** default backgrounds and reading surfaces
- **Indigo:** focus, links, secondary emphasis
- **Teal:** success and supportive accent only

### 5.3 Surface behavior

Default surface behavior changes from:

- frosted
- glowing
- layered

to:

- matte
- lightly elevated
- softly bordered

Use glass only when it communicates a special moment:

- auth hero
- celebratory overlay
- emotional highlight card

Do not use heavy glass for:

- every list row
- every CTA container
- every section shell

### 5.4 Contrast rules

Marketing and primary app surfaces should be readable at a glance:

- stronger text/background contrast
- quieter background treatments
- subtler gradients
- clear border definition on cards

## 6. Typography Rules

### 6.1 Hierarchy

The user should immediately know:

1. what this screen is about
2. what matters most
3. what to do next

### 6.2 Usage

- **Display:** reserved for hero or single emotional metric
- **Title / titleSm:** default for screen and section headers
- **Body:** utility copy
- **Caption:** metadata, labels, supporting hints

### 6.3 Tone

Copy must be:

- short
- specific
- low-pressure

Prefer:

- "Còn 3 ngày đến kỷ niệm"
- "Hôm nay bạn có thể nhắn một câu ngắn"
- "Thêm điều quan trọng để Em+ nhắc đúng lúc"

Avoid:

- poetic but vague headlines
- long emotional paragraphs
- multiple metaphors in one section

## 7. Layout System

### 7.1 Web

Target structure:

1. Hero: pain + promise + one CTA
2. 3-value explainer: remember / suggest / care
3. Use-case or benefit cards
4. Audience fit / trust / everyday scenarios
5. CTA repeat

Rules:

- no bento gallery for the primary feature story
- no stock-photo-heavy layout for core value
- sections should have generous vertical rhythm
- each section should communicate one idea only

### 7.2 Mobile home

Target structure:

1. Greeting
2. Important upcoming item
3. One suggested action today
4. Quick access to reminders / memory / timeline
5. Upcoming dates

Rules:

- reduce chrome
- reduce floating ornaments
- reduce dashboard feeling
- increase action clarity

Home should answer:

- what is important now?
- what should I do today?
- what should I not forget?

## 8. Component Rules

### 8.1 Buttons

- one visually dominant primary CTA per screen
- secondary buttons should look lighter and calmer
- avoid multiple equal-weight buttons in one row when decision urgency is low

### 8.2 Cards

Cards should feel:

- soft
- clean
- readable

Card rules:

- radius 20–28 stays
- border visible but light
- shadow soft and short
- background more matte than glossy
- decorative gradients only as a soft wash, never as the content itself

### 8.3 Badges & pills

Badges should communicate:

- timing
- priority
- context

Not decoration.

### 8.4 Icons

- outline or simple filled icons
- one accent color per card
- no sticker-like icon treatment unless it carries meaning

### 8.5 Motion

Keep:

- subtle reveal
- press feedback
- gentle number transitions

Reduce:

- parallax
- snowfall
- decorative looping motion on high-conversion surfaces
- animations that compete with reading

## 9. Page-Level Guidance

## 9.1 Landing page

The landing page should move from:

- "show everything Em+ can do"

to:

- "show why Em+ deserves attention in the first 10 seconds"

### Hero requirements

- problem-first headline
- one-line promise
- one primary CTA
- low-friction reassurance

### Feature section requirements

Only show the product's first believable wedge:

- important dates
- things to remember about partner
- gentle action suggestions
- reminder support

Do not lead with:

- budget
- realtime sync
- mood engines
- hidden future platform breadth

### Trust section requirements

Until real testimonials exist, prefer:

- audience-fit statements
- situations the product helps with
- privacy and clarity cues

Avoid fake-looking "perfect couple" quotes.

## 9.2 Mobile home

The home screen should become more like a calm checklist than a glossy dashboard.

### Required information order

1. greeting / context
2. important date or relationship status
3. one suggested action
4. upcoming reminders

### Content rules

- one hero message only
- if a card exists, it must help action or recall
- remove any decorative element that does not improve understanding

## Phase C — Follow-up screens

- onboarding
- login / register
- pairing
- reminders
- timeline list

### Phase C emphasis

The next design effort should focus on **mobile before web**.

Reason:

- Em+ becomes believable when daily utility feels clear inside the app
- mobile is where reminders, pairing, memory capture, and lightweight care actions actually happen
- if mobile still feels decorative or vague, the landing page promise will not hold up

Priority order:

1. onboarding
2. pairing
3. reminders
4. timeline list
5. login / register polish

## 11. Acceptance Criteria

The direction is working if:

- a first-time visitor understands the product in under 10 seconds
- the web landing feels more credible and less noisy
- the mobile home feels easier to scan
- each primary screen has one clear next action
- brand warmth remains intact without relying on heavy decoration

## 12. Non-goals

This direction does **not** mean:

- copying LookAway visually
- removing dark mode
- removing all emotional tone
- turning Em+ into a sterile productivity app

The target is:

> "soft enough to feel personal, clear enough to feel trustworthy"
